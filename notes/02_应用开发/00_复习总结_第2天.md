## 🎯 大模型基础 - 核心知识点速查

### 1. Transformer架构 ⭐⭐⭐⭐⭐

**核心面试题：**
> 什么是Transformer？它解决了什么问题？

**30秒回答：**
```
Transformer是一个让AI"一眼看全"整个句子的架构。
解决了RNN/LSTM的三大问题：
1. 无法并行计算（顺序处理）
2. 长距离依赖丢失
3. 只能单向处理

核心是自注意力机制 + Query-Key-Value
```

**关键公式：**
```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

---

### 2. 三种架构变体 ⭐⭐⭐⭐⭐

**快速对比：**

| 架构 | 代表模型 | 注意力方向 | 训练任务 | 适用场景 |
|------|---------|-----------|---------|---------|
| **Encoder-only** | BERT | 双向 | MLM（填空） | 理解任务：分类、NER、QA |
| **Decoder-only** | GPT | 单向 | CLM（预测下一个词）| 生成任务：对话、写作、代码 |
| **Encoder-Decoder** | T5 | 双向+单向 | Span Corruption | 理解+生成：翻译、摘要 |

**记忆口诀：**
```
BERT理解世界（看全部）
GPT创造世界（不偷看）
T5转换世界（翻译）
```

---

### 3. Tokenization（分词）⭐⭐⭐⭐

**核心概念：**
```
Subword（子词）= 介于字符和词之间

三大优势：
1. 解决生僻词（无OOV）
2. 词表可控（5万-10万）
3. 多语言通用
```

**BPE算法：**
```
1. 初始化为字符级
2. 统计最频繁字符对
3. 合并成新子词
4. 迭代到目标词表大小
```

---

### 4. 预训练与微调 ⭐⭐⭐⭐⭐

**核心对比：**

| 对比 | 预训练 | 微调 |
|------|--------|------|
| 目标 | 学习通用能力 | 适应特定任务 |
| 数据 | 数万亿token | 几千到几十万条 |
| 成本 | 数百万美元 | 几百到几千美元 |
| 时间 | 数周到数月 | 几小时到几天 |
| 参与者 | 只有大公司 | 任何人 |

**LoRA核心公式：**
```
W' = W + AB（低秩分解）
参数量：0.1%-1%
效果：90%+
```

---

### 5. 评估指标 ⭐⭐⭐⭐⭐

**Perplexity（困惑度）：**
```
定义：模型对测试数据的困惑程度
通常越低越好，但创造性任务需要适度困惑度
公式：exp(-1/N × log P)
```

**BLEU vs ROUGE：**
```
BLEU：精确率（模型输出有多少和参考答案重叠）
ROUGE：召回率（参考答案有多少被模型覆盖）
```

**关键洞察：**
```
Perplexity陷阱：
- 训练集低、测试集高 → 过拟合！
- 创造性任务需要适度Perplexity
```

---

### 6. Transformer优化技术 ⭐⭐⭐⭐⭐

**Flash Attention：**
```
核心思想：减少显存访问，不是优化算法
效果：速度快2-4倍，支持超长序列
```

**KV Cache：**
```
核心思想：缓存已计算的K、V向量
效果：速度提升20-25倍
代价：显存+30-40%
```

**量化：**
```
FP16 → INT8 → INT4
显存节省：50% → 75% → 87.5%
精度损失：<0.01% → <1% → 1-3%
```

---

### 7. MoE（Mixture of Experts）⭐⭐⭐⭐

**核心思想：**
```
稀疏激活：总参数47B，但每次只用2个专家（14B）

优势：知识容量大 + 计算效率高
挑战：专家塌缩、显存占用大、通信开销

例子：Mixtral 8x7B
```

---

### 8. 投机采样 ⭐⭐⭐⭐

**原理：**
```
小模型草拟K个token（实习生）
大模型并行验证这K个token（老板）
接受正确的，拒绝并重新生成错误的

加速公式：
加速比 ≈ 接受率 × 小模型加速倍数 / (1 + 小模型加速倍数)
```

---

### RAG系统 ⭐⭐⭐⭐⭐

**完整流程：**
```
1. 文档加载
2. 文档分割
3. 向量化
4. 存储
5. 检索（top-k）
6. 增强（构造提示词）
7. 生成
```

**RAG vs 微调：**
```
RAG = 开卷考试（给参考资料）
微调 = 闭卷考试（内化知识）

选择建议：
- 知识更新频繁 → RAG
- 需要特定风格 → 微调
- 最佳实践 → RAG + 微调
```

---

### Agent的ReAct模式 ⭐⭐⭐⭐⭐

**工作流程：**
```
用户输入
  ↓
Thought: 分析问题，决定下一步
  ↓
Action: 调用工具
  ↓
Observation: 获取工具结果
  ↓
Thought: 基于结果决定是否继续
  ↓
循环或返回结果
```

**关键洞察：**
```
Agent的"智能"来自于迭代优化
不是一次性完成，而是多次观察后决策
```

## 🔑 关键公式汇总

```
1. 自注意力机制：
   Attention(Q, K, V) = softmax(QK^T / √d_k) × V

2. LoRA：
   W' = W + AB（其中 A: n×r, B: r×n, r << n）

3. Perplexity：
   Perplexity = exp(-1/N × Σ log P(wi|context))

4. 投机采样加速比：
   加速比 ≈ 接受率 × 小模型加速倍数 / (1 + 小模型加速倍数)

5. BPE合并条件：
   选择最频繁的字符对进行合并
```

## 📌 面试高频追问

**追问1：为什么除以√d_k？**
```
防止点积过大导致softmax梯度消失
缩放后让梯度更稳定
```

**追问2：LoRA加在哪些层？**
```
通常加在attention层的Q和V投影矩阵
因为这些层对任务适应最关键
```

**追问3：如何选择LoRA的rank？**
```
r=8:  简单任务
r=16: 中等任务（最常用）
r=32: 复杂任务
r=64: 接近全量微调
```

**追问4：投机采样什么时候加速？**
```
接受率 > 50%时才加速
理想情况：接受率70%，小模型快10倍 → 加速3-4倍
```

**追问5：RAG的检索策略有哪些？**
```
1. 相似度检索
2. MMR（平衡相关性和多样性）
3. Self-Query（按元数据过滤）
```
