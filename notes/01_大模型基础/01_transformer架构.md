## 🎯 核心面试题

### Q1: 请用自己的话解释什么是Transformer，它解决了什么问题？

**答案要点：**
- Transformer是一个让AI模型"一眼看全"整个句子的架构
- 解决了RNN/LSTM的三大问题：无法并行、长距离依赖丢失、只能单向处理
- 核心是自注意力机制 + Query-Key-Value

---

### Q2: 什么是Multi-Head Attention？为什么需要多头？

**答案要点：**
- 单头注意力只能捕捉一种关系模式
- 多头同时关注不同类型的关系（语法、语义、指代等）
- 类比：多人同时从不同角度观察同一事物

---

### Q3: BERT、GPT、T5有什么区别？分别适用什么场景？

**答案要点：**
| 架构 | 特点 | 适用场景 | 例子 |
|------|------|---------|------|
| Encoder-only (BERT) | 双向理解 | 分类、NER、QA | 情感分析 |
| Decoder-only (GPT) | 单向生成 | 对话、写作、代码生成 | ChatGPT、Claude |
| Encoder-Decoder (T5) | 理解+生成 | 翻译、摘要 | 中英翻译 |

> 像给完全不懂技术的人解释一样，用最简单的语言描述

**我的解释：**

Transformer是一个让AI模型"一眼看全"整个句子的架构。

**之前的RNN/LSTM像读书：** 必须从第一页读起，逐字往后看，读到最后一页时可能已经忘了开头的内容。

**Transformer像看画：** 同时看到整个画面中的所有元素，直接理解每个元素与其他所有元素的关系。

**实现方式：**
1. 把每个词转换成向量（数字列表）
2. 计算词与词之间的相似度（余弦相似度）
3. 通过Query-Key-Value机制，让每个词"主动寻找"它相关的词
4. 根据相似度加权，让每个词吸收其他词的信息

## 🔑 核心知识点

### 1. Transformer的诞生背景

**RNN/LSTM的局限性：**
- 🔴 顺序处理，无法并行计算（训练慢）
- 🔴 长距离依赖信息会丢失（梯度消失/爆炸）
- 🔴 每个词只能看到"之前"的词（双向信息需要两次处理）

**Transformer论文：**
- 论文：《Attention Is All You Need》(2017)
- 作者：Vaswani et al. (Google)
- 核心思想：完全抛弃RNN，只用注意力机制

### 2. 核心组件

```
Transformer架构：
┌─────────────────────────────────────┐
│         Encoder (编码器)              │
│  ┌────────────────────────────┐     │
│  │ Multi-Head Attention       │     │
│  │ Feed Forward Network       │     │
│  └────────────────────────────┘     │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│         Decoder (解码器)              │
│  ┌────────────────────────────┐     │
│  │ Masked Multi-Head Attention│     │
│  │ Cross Attention            │     │
│  │ Feed Forward Network       │     │
│  └────────────────────────────┘     │
└─────────────────────────────────────┘
```

### 3. 自注意力机制 (Self-Attention)

**核心公式：**
```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

**步骤拆解：**
1. **线性变换**：输入X → 生成Q、K、V三个向量
2. **计算相似度**：Q × K^T（点积衡量相似度）
3. **缩放**：除以√d_k（防止梯度消失）
4. **归一化**：softmax转换为概率分布
5. **加权求和**：权重 × V → 得到最终的注意力输出

**举例：**
```
句子: "我 喜欢 学习 AI 因为 它 很有趣"

处理"它"时：
- Query("它") = "我在找一个指代对象"
- 与所有Key计算相似度
- 发现Key("AI")相似度最高 = 0.85
- 最终表示("它") ≈ 0.85 × Value("AI") + ...
```

### 4. 编码器-解码器结构

**Encoder（编码器）：**
- 作用：理解输入序列
- 输出：包含输入信息的语义表示
- 特点：可以看到整个输入序列

**Decoder（解码器）：**
- 作用：生成输出序列
- 两层注意力：
  - Masked Attention：只能看到"之前"的输出（防止偷看答案）
  - Cross Attention：关注Encoder的输出（获取输入信息）

### 5. 位置编码 (Positional Encoding)

**为什么需要位置编码？**
- Transformer同时处理所有词，本身不知道词的位置顺序
- 需要显式注入位置信息

**实现方式：**
- 使用正弦/余弦函数生成位置编码
- 不同维度用不同频率的波形
- 位置编码 + 词向量 → 输入Transformer

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

## 📌 重点记录

### 面试高频考点：

1. **Transformer vs RNN/LSTM**
   - 主要优势：并行计算 + 长距离依赖
   - 代价：计算量大（O(n²)的注意力复杂度）

2. **为什么除以√d_k？**
   - 防止点积过大导致softmax梯度消失
   - 缩放后让梯度更稳定

3. **Multi-Head Attention的作用**
   - 单头注意力只能捕捉一种关系模式
   - 多头让模型同时关注不同类型的关系（语法、语义、指代等）

4. **三种Transformer架构**
   - Encoder-only：BERT（理解任务）
   - Decoder-only：GPT（生成任务）
   - Encoder-Decoder：T5（翻译任务）
