## 🎯 核心面试题速查

### Q1: 什么是Transformer？它解决了什么问题？

**30秒回答：**
> Transformer是一个让AI模型"一眼看全"整个句子的架构。它解决了RNN/LSTM的三大问题：无法并行计算、长距离依赖信息丢失、只能单向处理。核心是自注意力机制和Query-Key-Value。

**关键词：** 自注意力、并行计算、长距离依赖、QKV

---

### Q2: 什么是Multi-Head Attention？为什么需要多头？

**30秒回答：**
> 单头注意力只能捕捉一种关系模式。Multi-Head Attention让模型同时关注不同类型的关系（语法、语义、指代等），就像多人从不同角度观察同一事物。

**关键词：** 多视角、关系模式、语法+语义

---

### Q3: BERT、GPT、T5有什么区别？

**快速对比表：**

| 架构 | 代表模型 | 注意力方向 | 训练任务 | 适用场景 | 例子 |
|------|---------|-----------|---------|---------|------|
| **Encoder-only** | BERT | 双向 | MLM（填空） | 理解任务 | 情感分析、NER、QA |
| **Decoder-only** | GPT | 单向 | CLM（预测下一个词）| 生成任务 | 对话、写作、代码 |
| **Encoder-Decoder** | T5 | 双向+单向 | Span Corruption | 理解+生成 | 翻译、摘要 |

**记忆口诀：**
- **BERT** = 双向理解（理解为王）
- **GPT** = 单向生成（生成至上）
- **T5** = 理解+生成（翻译专家）

---

### Q4: 为什么GPT要用Subword Tokenization？

**核心三要素：**
1. ✅ **解决生僻词** - 任何词都能拆成子词，没有OOV问题
2. ✅ **词表可控** - 5万-10万token，比词级（百万级）小得多
3. ✅ **多语言通用** - 同一套BPE算法适用于所有语言

**加分点：**
- 三种主流算法：BPE（GPT）、WordPiece（BERT）、Unigram LM（T5）
- 中文通常需要更多token（约1.5-2倍）

---

### Q5: 为什么BERT用双向，GPT用单向？

| 模型 | 方向 | 原因 | 应用 |
|------|------|------|------|
| **BERT** | 双向 | 理解任务需要看到完整上下文，双向理解更准确高效 | 分类、QA |
| **GPT** | 单向 | 生成任务不能"偷看"后面的词，单向才能真正的"创作" | 对话、写作 |

**一句话总结：**
> BERT要理解（看全部），GPT要生成（不偷看）

---

预训练 = 小学到大学的通识教育（学基础知识）

### Q7: LoRA的原理是什么？

**核心公式：**
```
W' = W + AB
  ↑    ↑  ↑
  |    |  └─ 低秩分解（r << n）
  |    └──── 原始权重不变
  └───────── 只训练A和B（不到1%的参数）
```

**为什么有效？**
- 微调时权重变化集中在低维空间
- 只需修改"主要变化方向"
- 类比：调照片亮度，一个参数就够了

**参数量对比：**
```
原始权重：4096 × 4096 = 16,777,216 参数
LoRA (r=16)：4096 × 16 + 16 × 4096 = 131,072 参数
比例：0.78% （不到1%！）
```

**四大优势：**
- 💰 成本低100倍
- ⚡ 训练快10倍
- 💾 存储小100倍
- 🎯 效果达90%+

### Q8: 场景题 - 如何选择微调方法？

**快速决策树：**

```
有没有数据？
├─ 没有 → 用预训练模型 + Prompt Engineering
└─ 有数据
    ├─ 数据量 < 1000 → Prompt Tuning
    ├─ 数据量 1000-10万 → LoRA（最推荐）⭐
    ├─ 数据量 10万-100万 → Full Fine-tuning 或 LoRA
    └─ 数据量 > 100万 + 大预算 → Full Fine-tuning
```

**常见场景：**
| 场景 | 方案 | 原因 |
|------|------|------|
| 创业公司，垂直领域助手 | LoRA | 预算有限，数据不多 |
| 大公司，打造通用产品 | 预训练或继续预训练 | 有资源，追求最好效果 |
| 快速验证想法 | Prompt Engineering | 无需训练，快速实验 |

## 📝 重点记忆公式

### 1. 自注意力机制
```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

### 2. LoRA
```
W' = W + AB（其中 A: n×r, B: r×n, r << n）
```

### 3. BPE算法流程
```
初始化字符 → 统计字符对频率 → 合并最频繁对 → 迭代到目标词表大小
```

## 🎯 面试高频追问及回答

### 追问1: 为什么除以√d_k？
**回答：** 防止点积过大导致softmax梯度消失，缩放后让梯度更稳定。

### 追问2: Transformer的复杂度是多少？
**回答：** O(n²)，其中n是序列长度。这是Transformer的主要缺点，长序列计算量大。

### 追问3: 中英文token差异？
**回答：** 中文约1.5-2字/token，英文约4字符/token，所以中文处理通常更贵。

### 追问4: LoRA加在哪些层？
**回答：** 通常加在attention层的Q和V投影矩阵（W_q和W_v），因为这些层对任务适应最关键。

### 追问5: 如何选择LoRA的rank？
**回答：**
- r=8: 简单任务（情感分类）
- r=16: 中等任务（对话、摘要）⭐最常用
- r=32: 复杂任务（代码生成）
- r=64: 接近全量微调效果
