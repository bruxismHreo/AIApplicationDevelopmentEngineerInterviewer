## 🎯 核心面试题

### Q1: 什么是预训练？什么是微调？为什么需要预训练？

**答案要点：**
- **预训练**：让模型在海量通用数据上学习"基础知识"（语法、世界知识、推理能力）
- **微调**：用特定任务数据让预训练模型"专业化"
- **为什么需要预训练**：从头训练需要海量数据和巨大成本，预训练后只需少量数据就能适应新任务

**类比：**
- 预训练 = 从小学到大学的通识教育
- 微调 = 大学毕业后的专业培训

---

### Q2: BERT和GPT的训练任务有什么不同？

**答案要点：**

| 模型 | 训练任务 | 注意力方向 | 目标 |
|------|---------|-----------|------|
| **BERT** | MLM（填空）、NSP（下一句预测） | 双向 | 理解上下文 |
| **GPT** | CLM（预测下一个词） | 单向 | 生成文本 |

**核心区别：**
- BERT同时看到所有词（双向理解）→ 适合分类、QA、NER
- GPT只看前面的词（单向生成）→ 适合写作、对话、代码

---

### Q3: LoRA的原理是什么？相比全量微调有什么优势？

**答案要点：**
- **原理**：低秩分解，W' = W + AB，其中A和B是小矩阵（r << n）
- **参数量**：只需训练0.1%-1%的参数
- **优势**：
  - 成本低100倍
  - 训练快10倍
  - 存储小100倍
  - 效果接近全量微调（90%+）

## 📝 我的理解（费曼学习法）

**预训练与微调的核心思想：**

就像培养一个医生：
1. **预训练**：先学通用知识（数学、语文、生物、化学...）
2. **微调**：再学专业课程（内科、外科、药学...）

如果不预训练直接微调：
- ❌ 就像让小学生直接学医，完全学不会

如果只预训练不微调：
- ❌ 就像通识教育毕业后直接当医生，不会治病

**LoRA的巧妙之处：**

想象你要调整一张照片的风格：
- **Full Fine-tuning**：修改每个像素 → 成本高
- **LoRA**：只调整几个关键参数（亮度、对比度、色调）→ 成本低，效果90%+

## 🔑 核心知识点

### 1. 预训练（Pre-training）

**目标：** 让模型学习通用能力

**训练数据：**
- 规模：数万亿token
- 来源：整个互联网（网页、书籍、代码、论文）
- 多语言：中英文为主，覆盖100+语言

**学到什么：**
```
1. 语言学知识
   - 语法规则（"我吃饭" vs "饭吃我"）
   - 语义理解（"高兴" ≈ "开心"）
   - 指代关系（"它"指代什么）

2. 世界知识
   - 事实知识（北京是中国首都）
   - 常识知识（冰会融化）
   - 领域知识（物理、历史、编程...）

3. 推理能力
   - 逻辑推理（如果A>B，B>C，那么A>C）
   - 因果关系（因为下雨，所以地湿）
   - 数学推理（1+1=2）

4. 上下文理解
   - 长距离依赖
   - 多轮对话
   - 文本连贯性
```

**预训练任务：**

**BERT:**
- **MLM (Masked Language Modeling)**
  ```
  输入：我喜欢吃[MASK]，因为[MASK]很甜。
  目标：预测[MASK] = "苹果"、"它"
  ```
- **NSP (Next Sentence Prediction)**
  ```
  句A：我今天去超市买菜。
  句B：买了西红柿和鸡蛋。
  目标：预测A和B是否连续
  ```

**GPT:**
- **CLM (Causal Language Modeling)**
  ```
  输入：我喜欢吃
  目标：预测下一个词（P(苹果) > P(汽车)）
  ```

### 2. 微调（Fine-tuning）

**目标：** 让预训练模型适应特定任务

**微调数据：**
- 规模：几千到几十万条
- 来源：任务相关数据集
- 质量要求：高质量、标注准确

**微调类型：**

| 类型 | 数据量 | 成本 | 效果 | 适用场景 |
|------|--------|------|------|---------|
| **Full Fine-tuning** | 10万+ | 极高 | 最好 | 大公司、关键任务 |
| **LoRA** | 1万-10万 | 中等 | 很好 | 中小公司、快速迭代 |
| **Prompt Tuning** | <1万 | 低 | 一般 | 简单任务、快速验证 |

### 3. LoRA详解

**核心公式：**
```
原始权重：W (n × n)

LoRA修改：
W' = W + ΔW
ΔW = A × B

其中：
A: n × r
B: r × n
r << n（rank可以是8、16、32...）
```

**参数量计算：**
```
假设 W 是 4096 × 4096 的矩阵
原始参数量 = 4096 × 4096 = 16,777,216

使用LoRA (r=16):
A: 4096 × 16 = 65,536
B: 16 × 4096 = 65,536
总参数 = 131,072

比例 = 131,072 / 16,777,216 = 0.78%
```

**为什么有效？**
1. 微调时权重变化通常集中在低维空间
2. 不需要修改所有方向，只需修改"主要变化方向"
3. 类比：调整照片亮度，一个参数就够了

**实施步骤：**
```python
# 1. 加载预训练模型
model = load_model("Qwen-7B")  # 7B参数

# 2. 添加LoRA
lora_config = LoraConfig(
    r=16,              # rank，控制适应能力
    target_modules=["q_proj", "v_proj"],  # 微调哪些层
    lora_alpha=32,     # scaling factor
)

model = get_peft_model(model, lora_config)

# 3. 训练
trainable_params = 0.24%  # 只有不到1%的参数可训练

# 4. 保存（只保存LoRA权重）
model.save_pretrained("my-lora")  # 约30MB
```

### 4. PEFT（参数高效微调）方法对比

| 方法 | 原理 | 参数量 | 效果 | 复杂度 | 推荐度 |
|------|------|--------|------|--------|--------|
| **LoRA** | 低秩分解 | 0.1%-1% | ⭐⭐⭐⭐⭐ | 低 | ⭐⭐⭐⭐⭐ |
| **QLoRA** | LoRA + 量化 | 0.1%-1% | ⭐⭐⭐⭐⭐ | 低 | ⭐⭐⭐⭐⭐ |
| **Adapter** | 插入小型网络 | 1%-5% | ⭐⭐⭐⭐ | 中 | ⭐⭐⭐ |
| **Prefix Tuning** | 优化前缀向量 | <0.1% | ⭐⭐⭐ | 高 | ⭐⭐ |
| **Prompt Tuning** | 只学提示词 | <0.1% | ⭐⭐ | 低 | ⭐⭐ |

**推荐：**
- 首选：**LoRA** 或 **QLoRA**（量化版，更省显存）
- 备选：Adapter（传统方法，效果也不错）
- 实验：Prompt Tuning（快速验证想法）

## 📌 重点记录

### 面试高频考点：

1. **预训练 vs 微调的本质区别**
   - 预训练：学习通用能力（数据量大、成本高）
   - 微调：适应特定任务（数据量小、成本低）

2. **为什么需要预训练？**
   - 从头训练成本太高（数百万美元）
   - 预训练模型已具备通用能力
   - 微调只需少量数据就能见效

3. **BERT为什么用双向？**
   - 理解任务需要看到完整上下文
   - 双向理解更准确、更高效

4. **GPT为什么用单向？**
   - 生成任务不能"偷看"后面的词
   - 单向才能真正的"创作"

5. **LoRA为什么有效？**
   - 微调权重变化集中在低维空间
   - 低秩分解能捕捉主要变化方向
   - 大幅降低参数量和成本

### 常见误区：

❌ 误区1：微调比预训练更重要
✅ 正解：预训练提供基础能力，微调只是适应任务

❌ 误区2：LoRA效果不如全量微调
✅ 正解：LoRA通常能达到全量微调90%+的效果

❌ 误区3：数据越多越好
✅ 正解：高质量的小数据集优于低质量的大数据集

❌ 误区4：微调后预训练能力会丢失
✅ 正解：LoRA等技术能很好保留预训练能力

### 实战建议：

**场景1：创业公司，预算有限，要做垂直领域助手**
- 方案：使用开源模型（Qwen、Llama）+ LoRA微调
- 数据：1万-5万条高质量对话
- 成本：约$500-2000
- 时间：1-3天

**场景2：大公司，要打造通用产品**
- 方案：从零预训练或继续预训练
- 数据：数万亿token
- 成本：数百万美元
- 时间：数月

**场景3：快速验证想法**
- 方案：Prompt Engineering + RAG
- 数据：无需训练数据
- 成本：几乎为0
- 时间：1小时-1天
