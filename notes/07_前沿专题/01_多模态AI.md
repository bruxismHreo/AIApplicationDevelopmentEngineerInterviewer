## 🎯 核心面试题

### Q1: CLIP 的核心思想是什么？它如何实现图文对齐？

**答案要点：**
- Contrastive Learning（对比学习）框架
- Image Encoder 和 Text Encoder 联合训练
- 使用对比损失将匹配的图文对拉近，不匹配的推远
- 零样本分类能力：将类别视为文本进行检索

---

### Q2: 如何设计一个图文检索系统？

**答案要点：**
- 离线阶段：CLIP 预训练，提取图片和文本的 embedding
- 索引阶段：将图片 embedding 存入向量数据库
- 检索阶段：查询文本 → embedding → 向量检索 → Top-K 图片
- 优化：Metadata 过滤、重排序、多路召回

---

### Q3: GPT-4V、Gemini Vision、Claude Vision 的主要区别？

**答案要点：**
- GPT-4V：OCR 能力强，理解复杂文档
- Gemini Vision：原生多模态，视频理解能力强
- Claude Vision：细节感知好，适合分析图表
- 选择依据：任务类型、成本、延迟要求

---

### Q4: 如何处理视频理解任务？

**答案要点：**
- Frame Sampling：均匀采样或关键帧提取
- Temporal Modeling：时序建模（3D CNN、Transformer）
- 多模态融合：视觉 + 音频 + 文本
- 实际工程：限制帧数控制成本，使用滑动窗口处理长视频

---

### Q5: 什么是 HyDE（Hypothetical Document Embeddings）？它在多模态场景中的应用？

**答案要点：**
- 查询扩展技术：用 LLM 生成假设性答案
- 假设答案 → embedding → 检索相似文档
- 多模态应用：文本查询 → 生成假设性图像描述 → 检索相关图像
- 提升 Recall：解决查询-文档语义不匹配问题

> CLIP 的核心价值在于"统一表示空间"——它将不同模态（图像、文本）映射到同一个向量空间中，使得跨模态检索成为可能。理解这一点是关键。

> 图文检索系统的设计要点：
> 1. 预训练模型选择（CLIP vs 自训练）
> 2. 向量数据库选型（内存 vs 磁盘）
> 3. 检索策略（纯向量 vs 混合检索）
> 4. 后处理（重排序、多样性）

## 🔑 核心知识点

### CLIP 原理详解

**架构组成：**
```
输入图像 ──→ Image Encoder (Vision Transformer) ──→ Image Embedding
输入文本 ──→ Text Encoder (Transformer) ──→ Text Embedding
                              ↓
                      Contrastive Loss
                              ↓
              匹配对拉近，不匹配对推远
```

**训练目标：**
- 对于 N 对图文样本，计算 N×N 相似度矩阵
- 对角线元素（匹配对）最大化
- 非对角线元素（不匹配对）最小化
- 使用对称的交叉熵损失

**零样本能力：**
- 分类任务：将类别标签转为文本描述（"A photo of {class}"）
- 计算查询图像与所有类别文本的相似度
- 选择最高相似度的类别

### 视觉理解模型对比

| 模型 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| **GPT-4V** | OCR 强、文档理解 | 成本高、延迟大 | 文档分析、图表理解 |
| **Gemini Vision** | 原生多模态、视频理解 | API 限制较多 | 视频分析、长上下文 |
| **Claude Vision** | 细节感知好 | 图像数量限制 | UI 分析、设计评审 |
| **Qwen-VL** | 中文友好、开源 | 性能略逊 GPT-4V | 中文场景、成本敏感 |

### 图文检索系统设计

**离线处理流程：**
```python
# 1. 图片 embedding 提取
for image in image_database:
    embedding = clip.encode_image(image)
    vector_db.insert(image_id, embedding, metadata)

# 2. 文本 embedding 提取（可选）
for text in text_database:
    embedding = clip.encode_text(text)
    vector_db.insert(text_id, embedding, metadata)
```

**在线检索流程：**
```python
# 1. 查询文本编码
query_embedding = clip.encode_text(user_query)

# 2. 向量检索
candidates = vector_db.search(query_embedding, top_k=100)

# 3. Metadata 过滤
filtered = apply_filters(candidates, filters)

# 4. 重排序（可选）
reranked = rerank_model.rerank(user_query, filtered)

# 5. 返回结果
return reranked[:top_k]
```

**优化策略：**
- **多路召回**：CLIP 检索 + BM25 检索融合
- **重排序**：使用 CrossEncoder（如 BGE-Reranker）
- **Metadata 过滤**：提前过滤类别、时间、来源等
- **缓存策略**：热门查询缓存结果

### 视频理解技术

**Frame Sampling 策略：**
- **均匀采样**：每 N 帧取一帧（简单但可能错过关键信息）
- **关键帧提取**：基于场景变化、运动检测（复杂但更精准）
- **自适应采样**：动态调整采样率（成本与质量平衡）

**时序建模方法：**
- **3D CNN**：I3D、SlowFast（计算成本高）
- **Video Transformer**：TimeSformer、ViViT（效果好但慢）
- **Two-Stream**：空间流 + 时间流（经典方法）
- **工程实践**：对于问答任务，采样 8-16 帧通常足够

### 多模态 Embedding 模型

| 模型 | 训练数据 | Embedding 维度 | 特点 |
|------|----------|----------------|------|
| **CLIP (OpenAI)** | 4亿图文对 | 512/768 | 基础模型，通用性强 |
| **OpenCLIP** | 多种数据集 | 512-1024 | 开源，可定制训练 |
| **AltCLIP** | 多语言 | 768 | 支持中文等多语言 |
| **Chinese-CLIP** | 中文图文 | 768 | 中文场景优化 |
| **BGE-Visual** | 图文检索 | 1024 | 中文开源，性能好 |

## 📌 重点记录

### 面试高频考点：
- CLIP 原理：对比学习、联合训练空间
- 图文检索：离线 embedding + 在线检索 + 优化策略
- 视觉模型对比：GPT-4V vs Gemini vs Claude
- 视频理解：Frame Sampling、时序建模

### 常见误区：
- 误认为 CLIP 只能做图文检索（实际可做零样本分类）
- 忽视视频理解的成本问题（采样数量影响 Token 消耗）
- 混淆多模态 Embedding 与纯文本 Embedding

### 工程实践要点：
- 图像 Embedding 预计算并存储
- 使用 Metadata 过滤减少检索范围
- 重排序能显著提升准确率
- 视频理解要控制采样帧数以控制成本
