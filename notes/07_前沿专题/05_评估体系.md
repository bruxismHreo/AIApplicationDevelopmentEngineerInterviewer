## 🎯 核心面试题

### Q1: 如何评估 RAG 系统的效果？有哪些核心指标？

**答案要点：**
- Faithfulness（忠实度）：答案是否基于检索的上下文
- Answer Relevancy（答案相关性）：答案是否回答了问题
- Context Precision（上下文精确度）：检索到的上下文是否相关
- Context Recall（上下文召回率）：是否检索到了所有相关上下文
- 工具：RAGAS、TruLens、DeepEval

---

### Q2: RAGAS 框架是如何工作的？如何使用它评估 RAG 系统？

**答案要点：**
- RAGAS：RAG Assessment Framework
- 核心流程：生成测试数据 → 运行 RAG 系统 → 计算指标
- 指标计算：使用 LLM 作为 Judge 评估
- 实现：提供 question、contexts、answer、ground_truth
- 输出：各项指标的分数和详细报告

---

### Q3: 如何构建 RAG 系统的评估数据集？

**答案要点：**
- 方式 1：人工标注（高质量但成本高）
- 方式 2：合成数据生成（使用 LLM 生成问题和答案）
- 方式 3：用户日志挖掘（真实场景数据）
- 数据集包含：question、contexts、ground_truth
- 需要覆盖：简单问答、复杂推理、多跳查询等场景

---

### Q4: 如何进行 A/B 测试来评估 LLM 应用？

**答案要点：**
- 线上 A/B 测试：流量分流（50/50 或 90/10）
- 指标定义：业务指标（转化率、满意度）+ 技术指标（延迟、成本）
- 统计显著性：确保结果可靠（p-value、置信区间）
- 工具：Experiment 平台、Feature Flag 系统
- 注意：控制变量、稳定后判断

---

### Q5: 真实场景评估和基准测试有什么区别？

**答案要点：**
- 基准测试：标准化数据集、离线评估、可复现
- 真实场景评估：真实用户数据、在线评估、反映实际效果
- 基准测试优势：可对比、易调试
- 真实场景优势：反映实际用户体验
- 最佳实践：先基准测试验证，再真实场景验证

## 🔑 核心知识点

### RAG 评估指标详解

**1. Faithfulness（忠实度）：**
```
定义：生成的答案是否基于检索的上下文

评分标准：
- 完全基于上下文：1.0
- 部分基于上下文：0.5
- 包含上下文外信息：0.0

计算方法（LLM as Judge）：
Prompt：检查答案是否包含上下文中没有的信息
输入：answer, contexts
输出：分数 0-1
```

**2. Answer Relevancy（答案相关性）：**
```
定义：答案是否直接回答了问题

评分标准：
- 直接相关：1.0
- 部分相关：0.5
- 无关或避答：0.0

计算方法：
基于答案生成反向问题，与原问题计算相似度
```

**3. Context Precision（上下文精确度）：**
```
定义：检索到的上下文有多少是相关的

公式：Precision = 相关上下文数 / 总检索上下文数

示例：
检索 10 个文档，其中 7 个相关
Context Precision = 7/10 = 0.7
```

**4. Context Recall（上下文召回率）：**
```
定义：检索到了多少相关文档

公式：Recall = 检索到的相关文档数 / 总相关文档数

示例：
总共有 20 个相关文档，检索到了 15 个
Context Recall = 15/20 = 0.75
```

### RAGAS 使用详解

**安装：**
```bash
pip install ragas
```

**基础使用：**
```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall
)

# 准备测试数据
test_data = {
    "question": [
        "什么是 RAG？",
        "Python 中如何读取 CSV？"
    ],
    "contexts": [
        ["RAG 是检索增强生成..."],
        ["使用 pandas.read_csv()..."]
    ],
    "answer": [
        "RAG 是检索增强生成，结合了检索和生成...",
        "使用 pandas 库的 read_csv 函数..."
    ],
    "ground_truth": [
        "RAG 是检索增强生成",
        "使用 pandas.read_csv() 函数"
    ]
}

# 运行评估
result = evaluate(
    test_data,
    metrics=[
        faithfulness,
        answer_relevancy,
        context_precision,
        context_recall
    ]
)

# 查看结果
print(result)
# 输出：
# {'faithfulness': 0.85, 'answer_relevancy': 0.92, ...}
```

**高级配置：**
```python
from langchain_openai import ChatOpenAI
from ragas.run_config import RunConfig

# 自定义 LLM 作为 Judge
evaluator_llm = ChatOpenAI(model="gpt-4")

result = evaluate(
    test_data,
    metrics=[faithfulness, answer_relevancy],
    run_config=RunConfig(
        llm=evaluator_llm,
        timeout=30  # 超时设置
    )
)

# 批量评估
for batch in dataloader(test_data, batch_size=10):
    result = evaluate(batch, metrics=metrics)
    save_result(result)
```

### 评估数据集构建

**方式 1：合成数据生成**
```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4")

def generate_synthetic_data(documents, num_samples=100):
    test_data = {
        "question": [],
        "contexts": [],
        "ground_truth": []
    }

    for _ in range(num_samples):
        # 随机选择文档
        doc = random.choice(documents)

        # 生成问题
        prompt = f"""
        基于以下文档生成一个高质量的问题：
        {doc}

        问题：
        """
        question = llm.predict(prompt)

        # 生成答案
        prompt = f"""
        基于以下文档回答问题：
        文档：{doc}
        问题：{question}

        答案：
        """
        ground_truth = llm.predict(prompt)

        test_data["question"].append(question)
        test_data["contexts"].append([doc])
        test_data["ground_truth"].append(ground_truth)

    return test_data
```

**方式 2：用户日志挖掘**
```python
def mine_user_logs(logs, min_rating=4):
    """
    从用户日志中挖掘高质量评估样本
    """
    test_data = {
        "question": [],
        "contexts": [],
        "answer": [],
        "ground_truth": []
    }

    for log in logs:
        # 过滤高质量交互
        if log["rating"] >= min_rating:
            test_data["question"].append(log["question"])
            test_data["contexts"].append(log["retrieved_docs"])
            test_data["answer"].append(log["generated_answer"])
            test_data["ground_truth"].append(log.get("user_answer", ""))

    return test_data
```

**数据集覆盖策略：**
```python
# 问题类型分类
question_types = {
    "factual": ["什么是...", "谁是...", "哪里..."],
    "procedural": ["如何...", "怎么...", "步骤..."],

# 确保各类型都有覆盖
def build_balanced_dataset(documents, samples_per_type=20):
    dataset = []
    for qtype, patterns in question_types.items():
```

### A/B 测试实践

**流量分流实现：**
```python
import hashlib

def ab_test(user_id, test_ratio=0.5):
    """
    根据 user_id 决定分配到 A 组还是 B 组
    """
    hash_value = int(hashlib.md5(user_id.encode()).hexdigest(), 16)
    return "A" if (hash_value % 100) < (test_ratio * 100) else "B"

# 使用
user_id = get_current_user()
group = ab_test(user_id, test_ratio=0.1)  # 10% 流量到 B 组

if group == "A":
    answer = rag_model_a(question)
else:
    answer = rag_model_b(question)
```

**指标收集：**
```python
from dataclasses import dataclass
from datetime import datetime

@dataclass
class ABTestMetrics:
    group: str
    user_id: str
    question: str
    answer: str
    latency: float
    user_rating: int = None
    timestamp: datetime = None

def collect_metrics(group, user_id, question, answer, latency):
    metrics = ABTestMetrics(
        group=group,
        user_id=user_id,
        question=question,
        answer=answer,
        latency=latency,
        timestamp=datetime.now()
    )

    # 存储到数据库
    db.insert("ab_test_metrics", metrics)

```

**统计分析：**
```python
from scipy import stats

def analyze_ab_test(metrics_a, metrics_b):
    """
    分析 A/B 测试结果，判断是否显著
    """
    # 计算平均评分
    mean_a = np.mean([m.user_rating for m in metrics_a])
    mean_b = np.mean([m.user_rating for m in metrics_b])

    # t 检验
    ratings_a = [m.user_rating for m in metrics_a]
    ratings_b = [m.user_rating for m in metrics_b]

    t_stat, p_value = stats.ttest_ind(ratings_a, ratings_b)

    # 判断显著性
    is_significant = p_value < 0.05

    return {
        "mean_a": mean_a,
        "mean_b": mean_b,
        "improvement": (mean_b - mean_a) / mean_a,
        "p_value": p_value,
        "is_significant": is_significant
    }
```

## 📌 重点记录

### 面试高频考点：
- RAG 评估指标：Faithfulness、Answer Relevancy、Context Precision、Context Recall
- RAGAS 使用：提供测试数据、运行评估、查看结果
- 数据集构建：合成数据 + 用户日志 + 人工标注
- A/B 测试：流量分流、指标收集、统计分析
- 线上监控：业务指标 + 技术指标 + 实时告警

### 常见误区：
- 只关注技术指标，忽视业务指标
- 评估数据集太小或不具代表性
- A/B 测试流量分配不合理（需要考虑统计显著性）
- 忽视线上监控的持续跟踪

### 工程实践要点：
- 建立自动化评估流水线，集成到 CI/CD
- 评估数据集需要覆盖各种场景和边界情况
- A/B 测试要先小流量验证，再逐步扩大
- 线上监控需要设置合理的告警阈值
- 定期更新评估数据集，跟上业务变化
- 评估结果要快速反馈到开发流程
