## 🎯 核心面试题

### Q1: 如何设计一个语义缓存系统来降低 LLM 调用成本？

**答案要点：**
- 缓存原理：相似查询复用之前的回答
- 实现方式：向量存储 + 相似度检索
- 缓存策略：余弦相似度阈值（如 >0.95 命中）
- 过期机制：TTL、LRU 淘汰
- 挑战：平衡准确率和缓存命中率

---

### Q2: Prompt Caching（GPT-4o、Claude）是如何工作的？如何利用它降低成本？

**答案要点：**
- 缓存静态 Prompt 部分（系统提示、固定上下文）
- API 支持缓存 Token，减少重复计费
- GPT-4o：通过 `cache_control` 参数启用
- Claude：自动缓存前 1000 个 Prompt Token
- 成本节约：长期对话、固定上下文场景可节省 50%+

---

### Q3: 如何实现模型路由（Model Routing）来优化成本？

**答案要点：**
- 根据任务复杂度选择不同成本的模型
- 简单任务：小模型（GPT-4o-mini、Llama 3 8B）
- 复杂任务：大模型（GPT-4o、Claude Opus）
- 路由策略：基于规则、基于分类器、基于 LLM 判断
- 成本优化：80% 任务用小模型，20% 用大模型

---

### Q4: 如何通过优化 Token 计数来降低成本？

**答案要点：**
- 输入优化：精简 Prompt、删除冗余上下文
- 输出优化：限制输出长度、使用结构化输出
- 计数监控：实时统计 Token 使用
- 切片优化：按需加载文档片段
- 工具：tiktoken（OpenAI）、tokenizers（HuggingFace）

---

### Q5: 小模型策略（SLM）有哪些？如何选择？

**答案要点：**
- SLM（Small Language Models）：Llama 3 8B、Mistral 7B、Phi-3
- 模型蒸馏：用大模型数据训练小模型
- 量化：FP16 → INT8 → INT4，减少内存和推理成本
- 选择依据：任务复杂度、延迟要求、成本预算
- 部署：本地部署、vLLM 推理加速

### 语义缓存详解

**工作原理：**
```
1. 用户查询 → Embedding
2. 检索缓存（向量相似度）
3. 相似度 > 阈值 → 返回缓存答案
4. 相似度 ≤ 阈值 → 调用 LLM → 存入缓存
```

**实现示例：**
```python
import numpy as np
from qdrant_client import QdrantClient

class SemanticCache:
    def __init__(self, threshold=0.95):
        self.client = QdrantClient(url="localhost:6333")
        self.threshold = threshold
        self.collection = "llm_cache"

    def get(self, query, query_embedding):
        # 检索相似查询
        results = self.client.search(
            collection_name=self.collection,
            query_vector=query_embedding,
            limit=1,
            score_threshold=self.threshold
        )

        if results:
            return results[0].payload["answer"]
        return None

    def set(self, query, query_embedding, answer):
        # 存入缓存
        self.client.upsert(
            collection_name=self.collection,
            points=[{
                "id": hash(query),
                "vector": query_embedding,
                "payload": {
                    "query": query,
                    "answer": answer,
                    "timestamp": time.time()
                }
            }]
        )

# 使用
cache = SemanticCache(threshold=0.95)

def llm_with_cache(query):
    query_embedding = embedding_model.encode(query)

    # 尝试从缓存获取
    cached_answer = cache.get(query, query_embedding)
    if cached_answer:
        return cached_answer

    # 调用 LLM
    answer = llm.generate(query)

    # 存入缓存
    cache.set(query, query_embedding, answer)

    return answer
```

**成本对比：**
```
场景：客服助手，系统提示 500 Token

不使用缓存：
- 每次请求：500 (系统) + 200 (用户) + 300 (输出) = 1000 Token
- 成本：1000 × 价格

使用缓存：
- 首次：1000 Token
- 后续：200 (用户) + 300 (输出) + 50 (缓存) = 550 Token
- 节约：(1000 - 550) / 1000 = 45%
```

### 模型路由详解

**路由策略：**
```python
class ModelRouter:
    def __init__(self):
        self.small_model = "gpt-4o-mini"  # 便宜
        self.large_model = "gpt-4o"       # 昂贵

    def route(self, query, context_length=0):
        # 策略 1：基于上下文长度
        if context_length < 1000:
            return self.small_model

        # 策略 2：基于查询复杂度
        complexity = self.estimate_complexity(query)
        if complexity < 0.5:
            return self.small_model
        else:
            return self.large_model

    def estimate_complexity(self, query):
        # 使用小模型判断复杂度
        prompt = f"""
        评估以下查询的复杂度（0-1之间）：
        {query}

        只返回数字。
        """
        result = self.small_model.generate(prompt)
        return float(result.strip())

# 使用
router = ModelRouter()

def generate(query, context):
    model = router.route(query, len(context))
    return model.generate(query, context)
```

**路由效果对比：**
```
场景：1000 个查询

无路由（全部用 GPT-4o）：
- 成本：1000 × $0.005 = $5

有路由（80% 用 mini，20% 用 GPT-4o）：
- 成本：800 × $0.0002 + 200 × $0.005 = $0.16 + $1 = $1.16
- 节约：($5 - $1.16) / $5 = 76.8%
```

## 📌 重点记录

### 面试高频考点：
- 语义缓存：向量存储 + 相似度检索 + TTL/LRU
- Prompt Caching：GPT-4o、Claude 的缓存机制
- 模型路由：根据任务复杂度选择模型
- Token 优化：精简 Prompt、限制输出、计数监控
- 小模型策略：Llama 3 8B、Mistral 7B、本地部署

### 常见误区：
- 认为缓存越多越好（需要平衡准确率和成本）
- 忽视模型路由的判断成本（判断本身也需要调用 LLM）
- 过度压缩 Prompt 导致效果下降
- 认为小模型一定便宜（本地部署有运维成本）

### 工程实践要点：
- 语义缓存阈值设置需要实验验证（0.9-0.98）
- Prompt Caching 在固定上下文场景效果最好
- 模型路由可以节省 50-80% 成本
- Token 优化是最基础的优化手段
- 小模型适合高并发、低延迟场景
- 量化可以显著降低部署成本，但会有精度损失
- 成本优化要避免过度优化影响用户体验
